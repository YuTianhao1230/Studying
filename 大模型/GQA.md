**GQA** 的全称是 **Grouped-Query Attention**（分组查询注意力）。它是一种在大型语言模型（LLM）中用于改进标准注意力机制的技术，旨在**在保持模型性能的同时，显著提高推理（Inference）速度并减少显存占用**。

为了理解 GQA，我们需要先看它产生的背景，以及它与另外两种注意力机制（MHA 和 MQA）的关系。

---

### 1. 为什么需要 GQA？（背景）

在 Transformer 模型（如 GPT、Llama）进行推理时，有一个关键的瓶颈叫做 **KV Cache**。
*   **KV Cache**：为了避免重复计算，模型会把之前生成过的 Token 的 Key（键）和 Value（值）向量存在显存里。
*   **问题**：随着上下文长度增加或 Batch Size（批大小）变大，KV Cache 会占用海量的显存，导致显存带宽成为瓶颈，推理速度变慢。

为了解决这个问题，研究者们演进出了三种方案：

#### A. MHA (Multi-Head Attention) - 多头注意力
*   **结构**：每个 Query (Q) 头都有对应的一组 Key (K) 和 Value (V) 头。
*   **特点**：效果最好，但 KV Cache 巨大，推理慢。

#### B. MQA (Multi-Query Attention) - 多查询注意力
*   **结构**：所有的 Q 头共用**唯一一组** K 和 V 头。
*   **特点**：KV Cache 极小，速度极快；但因为压缩太狠，模型表达能力下降，效果不如 MHA。

#### C. GQA (Grouped-Query Attention) - 分组查询注意力
*   **结构**：这是 MHA 和 MQA 的**折中方案**。将 Q 头分成若干组，每组 Q 头共用一组 K 和 V 头。
*   **特点**：速度接近 MQA，效果接近 MHA。

---

### 2. GQA 的核心原理

假设一个模型有 8 个 Query 头：
*   **在 MHA 中**：有 8 个 K 和 8 个 V（1:1 比例）。
*   **在 MQA 中**：只有 1 个 K 和 1 个 V（8:1 比例）。
*   **在 GQA 中**：我们可以把 8 个 Q 分成 2 组，每组 4 个 Q。每组共用 1 个 K 和 V。总共只有 2 个 K 和 2 个 V（4:1 比例）。

**直观对比图：**
```text
MHA: [Q1,K1,V1] [Q2,K2,V2] [Q3,K3,V3] [Q4,K4,V4]  (一一对应)
MQA: [Q1,Q2,Q3,Q4] -> [K1,V1]                    (全部共用)
GQA: [Q1,Q2]->[K1,V1] | [Q3,Q4]->[K2,V2]         (分组共用)
```

---

### 3. GQA 的主要优势

1.  **减少显存占用（KV Cache 优化）**：
    由于 K 和 V 的数量减少了（例如减少到原来的 1/4 或 1/8），存储 KV Cache 所需的显存大幅下降。
2.  **提高推理吞吐量（Speed Up）**：
    推理时的瓶颈往往不在于计算量，而在于从显存读取数据到计算核心的**带宽**。KV Cache 变小意味着读取的数据量变少，推理速度显著加快。
3.  **模型质量几乎无损**：
    实验证明，GQA 在性能上非常接近 MHA，远优于 MQA。它在“压缩”和“精度”之间找到了一个完美的平衡点。

---

### 4. 谁在用 GQA？

GQA 现在已经成为高性能大模型的**标配**。

*   **Llama 2 / Llama 3**：Llama 2 的 70B 版本使用了 GQA，而 Llama 3 的所有版本（8B, 70B, 400B）都全面采用了 GQA。
*   **Mistral / Mixtral**：Mistral 7B 和 Mixtral 8x7B 也使用了这种技术。
*   **其他**：像通义千问 (Qwen)、Gemma 等现代模型也纷纷采用 GQA。

### 总结
**GQA 是为了让大模型跑得更快、省更多显存而设计的。** 它通过让多个 Query 头共享一组 Key/Value 头，巧妙地解决了 MHA 太慢、MQA 效果不好的问题。
